Welcome to Banking Application

A comprehensive, modern banking application built with cutting-edge technologies. This platform provides secure authentication, account management, transaction processing, and real-time dashboard analytics for a complete banking experience.

Step-by-Step Installation

Step 1: Clone the Repository

Clone the project from GitHub to your local machine.

Terminal
git clone https://github.com/koke1997/bankarstvo.git
cd bankarstvo

Step 2: Install Frontend Dependencies

Install all required npm packages for the React frontend.

Terminal
npm install

Note: This will install React, TypeScript, Webpack, and all other frontend dependencies.

Step 3: Install Backend Dependencies

Install Python packages required for the Flask backend.

Terminal
pip install -r requirements.txt

Step 4: Configure Environment Variables

Set up your environment variables for database and API configuration.

Terminal
# Copy the example environment file
cp .env.example .env

# Edit .env with your configuration
# Configure database credentials, API keys, etc.

Step 5: Set Up the Database

Initialize the MySQL database and run migrations.

Terminal
# Create database
mysql -u root -p -e "CREATE DATABASE bankarstvo;"

# Run migrations
alembic upgrade head

Step 6: Start the Development Server

Launch both frontend and backend development servers.

Terminal
# Start the frontend development server
npm run dev

# In a separate terminal, start the backend server
python app.py

The application will be available at http://localhost:3000

Usage Guide

Learn how to use the banking application with practical examples and code snippets.

Getting Started

1. User Registration

First, register a new user account through the API or web interface.

JavaScript (Axios)
import axios from 'axios';

const registerUser = async () => {
    try {
        const response = await axios.post('http://localhost:5000/register', {
            username: 'john_doe',
            email: 'john@example.com',
            password: 'SecurePassword123!',
            full_name: 'John Doe'
        });
        
        console.log('Registration successful:', response.data);
        // Store the token for future requests
        localStorage.setItem('token', response.data.token);
    } catch (error) {
        console.error('Registration failed:', error.response.data);
    }
};

2. User Login

Authenticate with your credentials to receive an access token.

Python
import requests

def login_user(username, password):
    url = 'http://localhost:5000/login'
    payload = {
        'username': username,
        'password': password
    }
    
    response = requests.post(url, json=payload)
    
    if response.status_code == 200:
        data = response.json()
        token = data['token']
        print(f"Login successful! Token: {token}")
        return token
    else:
        print(f"Login failed: {response.json()}")
        return None

# Usage
token = login_user('john_doe', 'SecurePassword123!')

Working with Accounts

Creating a New Account

JavaScript (Fetch API)
const createAccount = async (token, accountData) => {
    const response = await fetch('http://localhost:5000/accounts', {
        method: 'POST',
        headers: {
            'Content-Type': 'application/json',
            'Authorization': `Bearer ${token}`
        },
        body: JSON.stringify(accountData)
    });
    
    const data = await response.json();
    return data;
};

// Usage
const newAccount = await createAccount(token, {
    account_type: 'checking',
    initial_balance: 1000.00,
    currency: 'USD'
});

const makeDeposit = async (accountId, amount) => {

Transfer Between Accounts

Python
def transfer_money(token, from_account, to_account, amount):
    url = 'http://localhost:5000/transactions/transfer'
    headers = {
        'Authorization': f'Bearer {token}',
        'Content-Type': 'application/json'
    }
    payload = {
        'from_account_id': from_account,
        'to_account_id': to_account,
        'amount': amount,
        'description': 'Account transfer'
    }
    
    response = requests.post(url, headers=headers, json=payload)
    
    if response.status_code == 201:
        print("Transfer successful!")
        return response.json()
    else:
        print(f"Transfer failed: {response.json()}")
        return None

# Usage
transfer_money(token, 1001, 1002, 300.00)

React Component Examples

Account Balance Component

React + TypeScript
import React, { useState, useEffect } from 'react';
import axios from 'axios';

interface Account {
    account_id: number;
    account_number: string;
    balance: number;
    account_type: string;
}

const AccountBalance: React.FC = () => {
    const [accounts, setAccounts] = useState<Account[]>([]);
    const [loading, setLoading] = useState(true);

    useEffect(() => {
        const fetchAccounts = async () => {
            try {
                const token = localStorage.getItem('token');
                const response = await axios.get('http://localhost:5000/accounts', {
                    headers: { 'Authorization': `Bearer ${token}` }
                });
                setAccounts(response.data.accounts);
            } catch (error) {
                console.error('Error fetching accounts:', error);
            } finally {
                setLoading(false);
            }
        };

        fetchAccounts();
    }, []);

    if (loading) return <div>Loading...</div>;

    return (
        <div className="accounts-list">
            {accounts.map(account => (
                <div key={account.account_id} className="account-card">
                    <h3>{account.account_type}</h3>
                    <p>Account: {account.account_number}</p>
                    <p>Balance: ${account.balance.toFixed(2)}</p>
                </div>
            ))}
        </div>
    );
};

Frequently Asked Questions

Find answers to common questions about the banking application.

Q: Is this application production-ready?
A: This is a demonstration/educational project showcasing modern web development practices. While it implements many production-ready features like authentication, validation, and security best practices, it would need additional auditing, testing, and compliance measures before being used in a real banking environment.
Q: What technologies are used in this project?
A: The frontend is built with React 18, TypeScript, and Tailwind CSS. The backend uses Python Flask with SQLAlchemy ORM. The database is MySQL 5.7+. The project also includes Docker and Kubernetes configurations for containerized deployment.
Q: How do I contribute to this project?
A: Contributions are welcome! Please fork the repository, create a feature branch, make your changes, and submit a pull request. Make sure to follow the code style guidelines and include tests for new features.
Q: Is the application secure?
A: The application implements several security features including JWT authentication, password hashing with bcrypt, SQL injection prevention through ORM, CORS configuration, and input validation. However, always conduct a thorough security audit before using any application in production.
Q: Can I use this project for learning purposes?
A: Absolutely! This project is designed to demonstrate modern web development practices and can be used as a learning resource. Feel free to explore the code, experiment with features, and adapt it for your own projects.
Q: What are the system requirements?
A: You need Node.js 18+, Python 3.9+, and MySQL 5.7+ to run the application locally. For Docker deployment, you only need Docker and Docker Compose installed. The application runs on Linux, macOS, and Windows.
Q: How do I deploy this to production?
A: The project includes Kubernetes configurations in the k8s/ directory for cloud deployment. You can also use the Docker Compose setup for simpler deployments. Make sure to configure environment variables properly and use a managed database service in production.
Q: Are there mobile apps available?
A: Currently, the application is a responsive web application that works well on mobile browsers. Native mobile applications are not yet available, but the REST API can be used to build mobile apps using React Native or other frameworks.
Q: Where can I report bugs or request features?
